Different between var, let and Const
1. Var

Var variable can be declare multiple times.Once declared, it can also be reassigned a value.The var variables have a global scope, which means that they can be accessed anywhere from the script.One particular trait of the var variable is that once declared in the global scope (e.g. at the very beginning of the script) it becomes available as a property in the global window objec


2. let

The let variables can only be declared once. If declared more than once, a Syntax Error will be thrown to the console.Same as var, let variable can also be reassigned a value.But when used in the global scope it won't be available in the global window object.let is block-scoped variable. This means that it's only accessible inside a single block, which is marked by a set of curly braces.let variables are also hoisted, but unlike for var, the variables are not initialized with a default value of undefined. A Reference Error is thrown instead


3. Const

The const variables can only be declared once. If declared more than once, a Syntax Error will be thrown to the console.Unlike var and let, the const variable cannot be reassigned a value.Just like the let variable, const is block-scoped variable. This means that it's only accessible inside a single block, which is marked by a set of curly braces.const variables are also hoisted, but unlike for var, the variables are not initialized with a default value of undefined. A Reference Error is thrown instead.

It is considered a good practice to use let and const most of the time and to avoid using var.

Different between Arrow Function and Regular Function
1.

This value inside a regular function is dynamic and depends on the invocation. But this inside the arrow function is bound lexically and equals to this of the outer function.

2.

arguments object inside the regular functions contains the list of arguments. The arrow function, on the opposite, doesn't define arguments (but you can easily access the arrow function arguments using a rest parameter ...args).

3.

If the arrow function has one expression, then the expression is returned implicitly, even without using the return keyword.

4.

you can define methods using the arrow function syntax inside classes. Fat arrow methods bind this value to the class instance.

Different between map, forEach, filter, find
1. map()

.map() executes the same code on every element in an array and returns a new array with the updated elements.

2. forEach()

.forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.

3. filter()

.filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.

4. find()

.find(), checks the criteria and when it meets it return the first element that truly meet the criteria and return a value not an array.
